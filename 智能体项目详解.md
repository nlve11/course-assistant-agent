## 一、项目概述

本智能体项目是一个基于命令行的课程问答助手。**核心功能**：接收用户输入的目标字符串，自主分析并选择合适的工具执行，最后返回结构化结果。

### 设计理念

```
用户输入 -> 安全校验 -> 关键词分析 -> 工具调用 -> LLM汇总 -> 返回结果
```

整个流程可以类比一个真实的团队协作：
- **guardrails.py** = 保安（检查每个人能不能进）
- **main.py + workflow.py** = 项目经理（分配任务）
- **tools.py** = 三个不同工种的员工（各司其职）
- **model_client.py** = 秘书（整理报告）
- **token_tracker.py** = 财务（记账）

```
   │
   └──→ src/model_client.py（check-model 命令）
```

### 3.2 一次完整调用的数据流

以下是一次完整的 `python main.py --goal "帮我搜索RAG相关笔记"` 执行过程：

```
第1步：main.py 解析参数
       ↓ 提取 goal = "帮我搜索RAG相关笔记"
       ↓ 调用 Workflow.run(goal)

第2步：guardrails.validate_input(goal)
       ↓ 检查是否为空 → 否
       ↓ 检查是否超过2000字 → 否
       ↓ 检查是否包含危险词 → 否
       ↓ 返回 safe=True

第3步：workflow 分析关键词
       ↓ 扫描: "搜索" → 匹配 search_notes
       ↓ 扫描: "RAG"  → 匹配 search_notes
       ↓ 决定 tools = ["search_notes"]

第4步：执行 search_notes(goal)
       ↓ 遍历 data/note/*.txt
       ↓ 找到 rag.txt（内容包含 "RAG"）
       ↓ 返回 {status:"ok", items:[{title:"rag", snippet:"RAG是..."}]}

第5步：model_client.chat(messages)
       ↓ 把 goal + 工具结果发给 GPT-4o-mini
       ↓ GPT 返回自然语言总结

第6步：token_tracker.record(...)
       ↓ 记录本次 token 用量
       ↓ 检查预算是否超限

第7步：返回完整 JSON
```

## 五、模块二：大模型客户端 model_client.py 详解

### 5.1 设计思路

这个模块是整个项目与"外面世界"的接口。它封装了 OpenAI 兼容的 API 调用，让其他模块不需要关心 HTTP 请求、Token 计算等细节。

### 5.2 逐行解析

```python
import os, time, logging
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()  # 加载 .env 文件中的环境变量
logger = logging.getLogger(__name__)
```

> **load_dotenv()**：读取项目根目录下的 .env 文件，把里面的变量加载到环境变量中。
> 这样后面用 os.getenv() 就能读到 API Key。

```python
class ModelClient:
    def __init__(self):
        self.api_key = os.getenv("OPENAI_API_KEY", "")
        self.base_url = os.getenv("OPENAI_BASE_URL",
                                  "https://api.openai.com/v1")
        self.model = os.getenv("MODEL_NAME", "gpt-4o-mini")
        self.max_cost = float(os.getenv("MAX_COST_YUAN", "1.0"))
        self.total_cost = 0.0
        self.client = None
        if self.api_key and self.api_key != "sk-your-key-here":
            self.client = OpenAI(
                api_key=self.api_key,
                base_url=self.base_url
            )
```

> **构造函数**：从环境变量读取配置。关键设计点：
> 1. 如果 OPENAI_API_KEY 没设置或者是默认的 "sk-your-key-here"，就不创建 OpenAI 客户端
> 2. 这样避免了启动时因为没有 API Key 就崩溃
> 3. 后续调用 is_ready() 可以判断客户端是否可用

```python
    def is_ready(self) -> bool:
        return self.client is not None
```

> 检查客户端是否已配置。如果返回 False，所有 API 调用都会返回提示信息而不是真正调模型。

```python
    def chat(self, messages: list, max_tokens=1024, temperature=0.7) -> dict:
        if not self.client:
            return {"content": "[Model not configured]",
                    "usage": {}, "cost": 0.0, "elapsed": 0.0}
        start = time.perf_counter()
        resp = self.client.chat.completions.create(
            model=self.model, messages=messages,
            max_tokens=max_tokens, temperature=temperature)
        elapsed = time.perf_counter() - start
        content = resp.choices[0].message.content
        u = resp.usage
        usage = {"prompt": u.prompt_tokens,
                 "completion": u.completion_tokens,
                 "total": u.total_tokens} if u else {}
        cost = usage.get("total", 0) * 0.00000015
        self.total_cost += cost
        logger.info("LLM call: model=%s tokens=%d cost=%.6f elapsed=%.2fs",
                     self.model, usage.get("total"), cost, elapsed)
        return {"content": content, "usage": usage,
                "cost": cost, "elapsed": elapsed}
```

> **chat() 方法**：核心 API 调用方法。
> 第3-4行：如果客户端没配置，直接返回提示信息，不报错。
> 第5-7行：计时 + 调用 OpenAI API。
> 第8-11行：从响应中提取 token 统计。
> 第12行：计算费用（0.15美元/百万token ≈ 0.00000015美元/token）。
> 第14-15行：记录日志，方便排查问题。

```python
    def check_model(self) -> dict:
        if not self.client:
            return {"status": "not_configured", "model": self.model}
        try:
            r = self.chat([{"role": "user", "content": "OK"}], max_tokens=5)
            return {"status": "ok", "model": self.model,
                    "tokens": r["usage"], "cost": r["cost"],
                    "elapsed": r["elapsed"]}
        except Exception as e:
            return {"status": "error", "model": self.model,
                    "message": str(e)}
```

> **check_model()**：发送一条极短的 "OK" 消息（只需几个 token），验证 API 连通性。
> 成功返回 status:ok，失败返回 error + 错误信息，未配置返回 not_configured。

## 六、模块三：工具实现 tools.py 详解

### 6.1 设计思路

三个工具都是"纯函数"——输入参数，返回结果，没有副作用（除了读文件）。这样设计的好处：
1. 易于测试：给定输入就能预测输出
2. 易于扩展：加一个新工具只需要再加一个函数
3. 与智能体流程解耦：workflow 不关心工具内部怎么实现

### 6.2 search_notes 逐行解析

```python
import os, glob

NOTE_DIR = os.path.join(
    os.path.dirname(os.path.dirname(__file__)),
    "data", "note"
)
```

> 知识库路径。`__file__` 是 tools.py 的路径，往上一级是 src/，再往上一级是项目根目录，然后拼接 data/note/。

```python
def search_notes(query: str, top_k: int = 3) -> dict:
    if not query or not query.strip():
        return {"status": "validation_error",
                "error": "query is empty"}
    if not os.path.isdir(NOTE_DIR):
        return {"status": "knowledge_base_missing",
                "error": "data/note directory not found"}
```

> **参数校验**：空查询返回 validation_error；目录不存在返回 knowledge_base_missing。
> 这两个错误码在 MCP 契约文档（docs/mcp_tool_contract.md）中有明确定义。

```python
    top_k = max(1, min(top_k, 5))
    q = query.lower()
    results = []
    for fpath in sorted(glob.glob(os.path.join(NOTE_DIR, "*.txt"))):
        with open(fpath, "r", encoding="utf-8") as f:
            content = f.read()
        title = os.path.splitext(os.path.basename(fpath))[0]
        if q in content.lower() or q in title.lower():
            snippet = content[:300].replace("\n", " ")
            results.append({
                "title": title,
                "snippet": snippet,
                "source": fpath
            })
    return {"status": "ok", "items": results[:top_k],
            "total": len(results)}
```

> **核心搜索逻辑**：
> 第2行：`max(1, min(top_k, 5))` 确保 top_k 在 1 到 5 之间。
> 第3-11行：遍历 data/note/ 下所有 .txt 文件。用 `query.lower()` 做大小写不敏感匹配。
> 第12-13行：匹配成功则截取前300字作为摘要，存入结果列表。
> 第14行：返回时用 `results[:top_k]` 限制最大返回数量。

```python
    if len(answer) >= 10:
        score += 2
    else:
        feedback.append("Too short")
    if rubric and rubric.lower() in answer.lower():
        score += 3
        feedback.append("Covers rubric")
```

### 6.4 code_check 逐行解析

```python
def code_check(code: str, language: str = "python") -> dict:
    if not code or not code.strip():
        return {"status": "validation_error",
                "error": "code is empty"}
    issues = []
    if language == "python":
        if "import " not in code and "def " not in code:
            issues.append("No imports or function defs")
        if code.count("(") != code.count(")"):
            issues.append("Unmatched parentheses")
        if len(code.split("\n")) > 50:
            issues.append("Code exceeds 50 lines")
    return {"status": "ok", "issues": issues,
            "issue_count": len(issues),
            "line_count": len(code.split("\n"))}
```

> **代码静态检查**：
> - 检查是否有 import 或 def（判断是否为有效代码）
> - 检查括号数量是否匹配（避免语法错误）
> - 检查是否超过50行（提示是否考虑重构）
> 注意：这只是非常基础的检查，远不及 pylint/flake8 等专业工具。设计目的是"演示工具调用的概念"。

## 七、模块四：安全护栏 guardrails.py 详解

### 7.1 设计思路

安全护栏是整个项目的第一道防线。它确保：
1. 恶意输入不会进入系统
2. 文件访问不会越界
3. 敏感信息不会出现在日志中
4. 只有允许的工具才能被调用

### 7.2 逐行解析

```python
import re, os

PROJECT_ROOT = os.path.dirname(os.path.dirname(__file__))
ALLOWED_PATHS = [os.path.join(PROJECT_ROOT, "data")]
```

> 路径限制的基础：只允许访问项目 data/ 目录。任何试图读取其他路径的操作都会被拒绝。

```python
SENSITIVE_PATTERNS = [
    "password", "token", "secret", "api_key", "sk-"
]
MAX_INPUT_LENGTH = 2000
BLOCKED_KEYWORDS = [
    "delete", "rm -rf", "shutdown", "format",
    "drop table", "ignore all instructions",
    "system password"
]
```

> **预定义的敏感词和危险词列表**：
> - SENSITIVE_PATTERNS：日志脱敏时用，匹配这些词后面的值会被替换为 ***
> - BLOCKED_KEYWORDS：输入校验时用，包含这些词的输入会被直接拒绝

#### 防护①：validate_input()

```python
def validate_input(text: str) -> dict:
    """Check user input for safety."""
    if not text or not text.strip():
        return {"safe": False, "reason": "Input is empty"}
    if len(text) > MAX_INPUT_LENGTH:
        return {"safe": False,
                "reason": f"Input exceeds {MAX_INPUT_LENGTH} chars"}
    lower = text.lower()
    for kw in BLOCKED_KEYWORDS:
        if kw in lower:
            return {"safe": False,
                    "reason": f"Blocked keyword: {kw}"}
    return {"safe": True, "reason": "ok"}
```

> **校验顺序**：空检查 → 长度检查 → 危险词检查。
> 注意 `text.lower()`：大小写不敏感匹配，"Delete"和"delete"都能被拦截。

**校验示例**：

| 输入 | 拦截原因 |
|------|---------|
| "" | Input is empty |
| "A" × 2001 | Input exceeds 2000 chars |
| "delete all files" | Blocked keyword: delete |
| "帮我搜索RAG" | 通过（safe=True） |

#### 防护②：validate_path()

```python
def validate_path(filepath: str) -> dict:
    abs_path = os.path.abspath(
        os.path.join(PROJECT_ROOT, filepath))
    for allowed in ALLOWED_PATHS:
        if abs_path.startswith(allowed):
            return {"safe": True, "path": abs_path}
    return {"safe": False,
            "reason": f"Path not allowed: {filepath}"}
```

> **路径限制逻辑**：
> 1. 把相对路径转换为绝对路径
> 2. 检查是否以 data/ 开头
> 3. 是则放行，否则拒绝

#### 防护③：sanitize_log()

```python
def sanitize_log(text: str) -> str:
    result = text
    for pat in SENSITIVE_PATTERNS:
        result = re.sub(
            pat + r"['\"]?[=:]['\"]?\S+",
            pat + "=***", result,
            flags=re.IGNORECASE
        )
    result = re.sub(r"sk-[A-Za-z0-9]{10,}",
                    "sk-***", result)
    return result
```

> **日志脱敏逻辑**：
> 第3-5行：匹配 "password=xxx"、"token:xxx" 这类模式，替换为 "password=***"
> 第6行：额外匹配 sk-xxx 格式的 API Key（至少10个字符）

#### 防护④：check_tool_allowed()

```python
def check_tool_allowed(tool_name: str) -> bool:
    return tool_name in {
        "search_notes",
        "evaluate_answer",
        "code_check"
    }
```

> **白名单机制**：只有显式列出的工具才能被调用。如果某个新工具不在白名单中，workflow 会记录 "tool_blocked" 步骤并跳过。

## 八、模块五：词元追踪 token_tracker.py 详解

### 8.1 设计思路

每次调用大模型都会消耗 token，而 token 等于钱。这个模块负责：
1. 记录每次调用的 token 数
2. 计算每次调用的费用
3. 累计总费用，检查是否超预算
4. 把数据导出为 JSON 日志文件

### 8.2 逐行解析

```python
    "logs"
)
```

> 初始化空的记录列表和累计数据。

```python
    def record(self, action: str, usage: dict, cost: float):
        entry = {
            "action": action,
            "prompt_tokens": usage.get("prompt", 0),
            "completion_tokens": usage.get("completion", 0),
            "total_tokens": usage.get("total", 0),
            "cost": round(cost, 8),
            "timestamp": time.strftime(
                "%Y-%m-%d %H:%M:%S"),
        }
        self.entries.append(entry)
        self.total_prompt += entry["prompt_tokens"]
        self.total_completion += entry["completion_tokens"]
        self.total_cost += cost
```

> **record() 方法**：记录一次 LLM 调用的数据。
> `round(cost, 8)`：费用保留 8 位小数（因为每次调用可能只花 $0.00002）。

```python
    def summary(self) -> dict:
        return {
            "total_entries": len(self.entries),
            "total_prompt_tokens": self.total_prompt,
            "total_completion_tokens":
                self.total_completion,
            "total_tokens":
                self.total_prompt + self.total_completion,
            "total_cost_yuan":
                round(self.total_cost * 7.2, 4),
            "total_cost_usd":
                round(self.total_cost, 6),
        }
```

> **summary() 方法**：汇总统计数据。
> `total_cost * 7.2`：把美元按约 7.2 的汇率换算为人民币。

```python
    def dump_log(self, session_id: str):
        os.makedirs(LOG_DIR, exist_ok=True)
        path = os.path.join(
            LOG_DIR, f"session_{session_id}.json")
        data = {
            "session_id": session_id,
            "entries": self.entries,
            "summary": self.summary()
        }
        with open(path, "w", encoding="utf-8") as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
        return path
```

> **dump_log() 方法**：把本次会话的所有 token 记录导出为 JSON 文件，方便后续查看和分析。

```python
    def budget_remaining(self) -> float:
        return max(0.0, 1.0 - self.total_cost * 7.2)

    def budget_exceeded(self) -> bool:
        return self.total_cost * 7.2 >= 1.0
```

> **预算控制**：
> - budget_remaining()：还剩多少钱（不能为负数）
> - budget_exceeded()：是否已超预算（超过1元返回 True）
> workflow 每次执行完 LLM 调用后检查这个值，超了就停止。

### 9.1 设计思路

workflow.py 是整个项目的"大脑"。它把前面所有模块串联起来，形成完整的执行流程。

### 9.2 逐行解析

```python
import logging, uuid
from .model_client import ModelClient
from .tools import search_notes, evaluate_answer, code_check
from .guardrails import validate_input, check_tool_allowed, sanitize_log
from .token_tracker import TokenTracker

logger = logging.getLogger(__name__)

TOOL_MAP = {
    "search_notes": search_notes,
    "evaluate_answer": evaluate_answer,

```

> **工具注册表**：一个字典，把工具名称映射到对应的函数。
> 如果要加新工具，只需要三步：
> 1. 在 tools.py 里写新函数
> 2. 在这里添加到 TOOL_MAP
> 3. 在 guardrails.py 的 check_tool_allowed 里加进白名单

```python
class Workflow:
    def __init__(self, model_client: ModelClient,
                 token_tracker: TokenTracker):
        self.model = model_client
        self.tracker = token_tracker
```

> **构造函数**：接收外部传入的 ModelClient 和 TokenTracker 实例。

```python
    def run(self, goal: str) -> dict:
        sid = uuid.uuid4().hex[:8]
        steps = []
```

> **run() 方法**：核心方法。`uuid.uuid4().hex[:8]` 生成一个8位会话ID。

```python
        # 步骤1：安全校验
        guard = validate_input(goal)
        if not guard["safe"]:
            return {"session_id": sid,
                    "status": "rejected",
                    "reason": guard["reason"],
                    "steps": steps,
                    "token_summary": self.tracker.summary()}
        steps.append({"step": "validated",
                      "goal": sanitize_log(goal)})
```

> **安全校验**：如果不通过，直接返回 rejected 状态和拒绝原因。注意这里用 sanitize_log 对 goal 做了脱敏再记录。

```python
        # 步骤2：关键词分析
        gl = goal.lower()
        tools = []
        if any(k in gl for k in ["search", "查找", "搜索",
                                  "note", "rag"]):
            tools.append("search_notes")
        if any(k in gl for k in ["eval", "评估", "评价",
                                  "评分"]):
            tools.append("evaluate_answer")
        if any(k in gl for k in ["check", "检查", "code",
                                  "代码"]):
            tools.append("code_check")
        if not tools:
            tools = ["search_notes"]  # 默认兜底
        steps.append({"step": "plan", "tools": tools})
```

> **关键词匹配**：用 any() 检查目标中是否包含特定关键词。
> 可以同时匹配多个工具（比如 "搜索并评估RAG的内容" 会同时调 search_notes 和 evaluate_answer）。

```python
        # 步骤3：执行工具
        results = {}
        for tname in tools:
            if not check_tool_allowed(tname):
                steps.append({"step": "blocked",
                              "tool": tname})
                continue
            fn = TOOL_MAP.get(tname)
            if not fn:
                continue
            try:
                if tname == "search_notes":
                    r = fn(goal)
                elif tname == "evaluate_answer":
                    r = fn(goal, f"Answer about: {goal}")
                else:
                    r = fn("def foo(): pass")
                results[tname] = {"status": r.get("status")}
                steps.append({"step": "tool_ok",
                              "tool": tname,
                              "status": r.get("status")})
            except Exception as e:
                results[tname] = {"status": "error"}
                steps.append({"step": "tool_error",
                              "tool": tname,
                              "error": str(e)})
```

> **工具执行**：
> 第3-5行：白名单检查（工具被注册但不一定在白名单里）
> 第6-8行：从 TOOL_MAP 获取函数
> 第10-17行：根据工具名称调用不同参数
> 第19-22行：异常处理，记录错误信息

```python
        # 步骤4：LLM 汇总
        msgs = [
            {"role": "system",
             "content": "Summarize tool results for the user."},
            {"role": "user",
             "content": f"Goal: {goal}\nResults: {results}"},
        ]
        llm = self.model.chat(msgs)
        if llm["usage"]:
            self.tracker.record("response",
                                llm["usage"], llm["cost"])
        steps.append({"step": "responded",
                      "tokens": llm["usage"].get("total", 0)})
        self.tracker.dump_log(sid)
```

> **LLM 汇总**：把目标和工具结果拼成一条消息发给大模型。
> 第9行：记录本次 token 用量。
> 第12行：导出日志文件。

```python
        # 步骤5：预算检查
        if self.tracker.budget_exceeded():
            return {"session_id": sid,
                    "status": "budget_exceeded",
                    "steps": steps,
                    "token_summary": self.tracker.summary()}
```

> **预算检查**：如果累计费用超过 1 元，停止并返回预算超限状态。

```python
        # 步骤6：返回结果
        return {
            "session_id": sid,
            "status": "completed",
            "answer": llm["content"],
            "steps": steps,
            "tool_results": results,
            "token_summary": self.tracker.summary(),
        }
```

> **正常返回**：包含完整执行记录的结构化 JSON。
